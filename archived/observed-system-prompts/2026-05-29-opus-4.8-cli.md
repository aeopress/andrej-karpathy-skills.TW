## What this is — and the headline finding

Opus 4.8 shipped on 2026-05-28 with a **"lean system prompt" as the default** for all models except Haiku, Sonnet, and Opus 4.7-and-earlier (confirmed independently by [Simon Willison](https://simonwillison.net/2026/May/28/claude-opus-4-8/) and [Unite.AI](https://www.unite.ai/what-opus-4-8-changes-for-anyone-running-agents-on-claude/)).

**Headline finding for this repo:** the eight verbatim guardrail lines that the [README mapping table](../../README.md#which-v1-rules-ended-up-where) quotes from the Opus 4.7 prompt are **gone** from the 4.8 lean prompt. The verbose `# Doing tasks` (13 bullets) and `# Executing actions with care` (multiple paragraphs) sections were compressed into a 5-bullet `# Harness` block plus two short prose paragraphs.

This does **not** weaken the repo's thesis — it sharpens it. See [§ What this means for the thesis](#what-this-means-for-the-thesis) below.

## Caveat — self-observation

This snapshot is the agent's transcription of its **own** Opus 4.8 session context. Unlike a CLI debug dump, it is mediated by the model's own reading of its prompt — treat it as a good-faith observation, not a byte-exact capture. Confidence is nonetheless high because:

1. The compression matches Anthropic's publicly announced "lean system prompt default for 4.8" exactly.
2. The 4.7 companion snapshot was independently MD5-verified across a fresh session (see its re-verification log), establishing the methodology.

To verify yourself: run `claude --model claude-opus-4-8` in an empty dir with no `~/.claude/CLAUDE.md`, and ask the model to repeat its system prompt verbatim. Differences are expected over CLI versions.

## 4.7 → 4.8 diff: where did the guardrails go?

The eight lines the README table quotes from 4.7, and their fate in 4.8:

| 4.7 verbatim quote | 4.8 lean prompt |
|---|---|
| "Don't add features, refactor, or introduce abstractions beyond what the task requires" | **Removed** — no equivalent line |
| "Three similar lines is better than a premature abstraction" | **Removed** |
| "Don't design for hypothetical future requirements" | **Removed** |
| "Don't add error handling, fallbacks, or validation for scenarios that can't happen" | **Removed** |
| "A bug fix doesn't need surrounding cleanup; a one-shot operation doesn't need a helper" | **Removed** |
| "Avoid backwards-compatibility hacks like renaming unused _vars..." | **Removed** |
| "For exploratory questions, respond in 2-3 sentences with a recommendation and the main tradeoff" | **Removed** |
| "Match the scope of your actions to what was actually requested" | **Removed** |

What 4.8 keeps, in compressed form:

| 4.8 lean line | Compresses which 4.7 content |
|---|---|
| "Write code that reads like the surrounding code: match its comment density, naming, and idiom." | The 4.7 comment-policy bullets + style-matching, collapsed to one line |
| "For actions that are hard to reverse or outward-facing, confirm first..." | The entire 4.7 `# Executing actions with care` section (~3 paragraphs) |
| "Report outcomes faithfully: if tests fail, say so..." | **New emphasis** — matches Anthropic's stated 4.8 "honesty" theme ("4x less likely to let a code flaw pass") |

**Structural change**: 4.7 had `# System / # Doing tasks / # Executing actions with care / # Using your tools / # Tone and style / # Text output / # Session-specific guidance`. 4.8 collapses all of that into `# Harness` (5 bullets) + 2 prose paragraphs.

## What this means for the thesis

The repo's 4.7-era argument was: *"v1's guardrails are already in the system prompt verbatim, so restating them dilutes signal."* In 4.8 those guardrails are no longer in the prompt text. Two readings:

- **Reading A (supported by Anthropic's own numbers):** the guardrails moved from the prompt into post-training (model weights). Anthropic reports 4.8 is "~4x less likely than Opus 4.7 to let a code flaw pass" with "sharper judgement." The *behavior* is stronger; it just no longer needs to be spelled out in the prompt. Under this reading the thesis holds and arguably strengthens — a leaner prompt means every line you add in `CLAUDE.md` competes against less, so restating now-internalized behavior is still wasted signal, and a 19-line file is even easier to keep clean.
- **Reading B (the adversarial case):** if a guardrail is neither in the prompt nor reliably internalized, v1's fuller rules might "become useful again" on 4.8.

**Only a behavioral A/B test on 4.8 can decide between A and B.** The original [N=40 A/B](../../EXPERIMENT.md) ran on Opus 4.7. Until a 4.8 re-run exists, this repo treats Reading A as the *likely* but not *proven* case, and says so.

## Honest status of the README mapping table

The mapping table's third column ("verbatim line in Opus 4.7 system prompt") remains **factually correct for 4.7** — that snapshot is real and independently verified. It is **no longer literally true for 4.8**, where the lean prompt has dropped these lines. The README now annotates this explicitly rather than silently implying the quotes still live in the current default prompt.
